# Required Skills Identified

## What this means
Skills identification maps the technical and domain skills needed to successfully deliver this initiative against the current team's capabilities. This surfaces skill gaps that need to be filled through hiring, training, contracting, or pairing.

## Why it matters
Teams assigned to work requiring skills they don't have will either produce poor-quality output or drastically underperform estimates. Identifying skill gaps early allows time to hire or bring in contractors, arrange training, or adjust the approach to use technologies the team already knows well. Discovering a critical skill gap mid-project is one of the most common causes of timeline slippage.

## How to complete this
- List the technical skills required by this initiative (specific technologies, frameworks, patterns, domain knowledge)
- Assess the team's proficiency level in each required skill
- Identify gaps where current proficiency is below what the project requires
- For each gap, determine the resolution strategy: training, pairing with an expert, contracting a specialist, or redesigning to use familiar technology
- Include domain knowledge gaps — not just technical skills, but business/product domain understanding

## Definition of done
- [ ] Required skills identified for all major components
- [ ] Team proficiency assessed against each requirement
- [ ] Skill gaps documented
- [ ] Resolution strategy defined for each gap
